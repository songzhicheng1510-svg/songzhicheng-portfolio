Text to Massing

A human-AI workflow for 3D massing design

01 Problem

自然语言设计任务很难直接变成稳定、可复查的建筑体量生成过程。

Problem

项目问题

早期办公建筑体量设计依赖人工经验和反复调整，自然语言任务书很难稳定转译为可复现、可审计的几何生成过程。

Method

方法流程

  1. 将任务书、规范和设计策略拆解为 Objectives、Constraints、Priorities、Examples、Inspection 五类 Prompt Pack。
  2. 用 GPT-5 Thinking 生成 Rhino Python 代码，并通过 corridor-first 组织、功能块合并、指标监测和错误修复推进迭代。
  3. 围绕 25 个办公建筑体量场景评估 Merge Rate、Programme Ratio Deviation 与直接基线模型差异。

Output

成果价值

形成从自然语言 brief 到可运行体量脚本的 Human-AI Workflow，并把设计知识外化为可检查、可复用的结构化规则。
